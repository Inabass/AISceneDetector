# Training Pipeline

## 学習動画管理

学習動画はアップロード時に SHA256、ffprobe メタデータ、ファイルサイズ、拡張子、登録種別を保存する。正例動画は抽出したいシーンのみを含む前提で扱う。負例動画は似ているが抽出したくないシーン、または通常背景を含む動画として扱う。

## 正例・負例

正例特徴量はモデルの中心表現を作る。負例特徴量は誤検出を抑えるために利用する。負例が未登録の場合でもモデル作成は可能だが、UI と評価結果で「負例なし」と明示する。

## フレーム抽出

OpenCV VideoCapture または ffmpeg 由来のフレーム抽出を利用する。フレーム間隔は秒単位または N フレーム単位で指定できる。長時間動画では全フレームをメモリに保持せず、バッチ単位で読み出す。

## Feature 生成

抽出フレームは RGB 変換、リサイズ、正規化を行い、OpenCLIP に入力する。GPU バッチサイズは設定値から開始し、OOM 時は半減して再試行する。特徴量は L2 正規化し、float16 保存を基本、必要に応じて float32 を許可する。

## Feature 保存

特徴量ファイルは `data/features/{model_id}/{version_id}/` に保存する。DB にはファイルパス、shape、dtype、フレーム範囲、生成ジョブ ID を記録する。途中保存により、キャンセルやエラー時も処理済み範囲を確認できる。

## 学習ジョブ

学習は Background Job として実行する。Job は `queued`、`running`、`cancel_requested`、`cancelled`、`failed`、`succeeded` を持つ。進捗は動画数、抽出フレーム数、特徴量生成済み件数、モデル保存段階で更新する。

## モデル生成

モデル生成では、正例特徴量の代表ベクトル、クラスタ、負例統計、しきい値初期値、評価メタデータを作成する。分類器を生成できるだけの負例がない場合は、Matcher のみで有効なモデルを作る。

## 途中保存

長時間処理では動画単位、チャンク単位、特徴量ファイル単位で checkpoint を保存する。再試行時は同一 SHA256、同一抽出設定の完了済み Feature を再利用できる。

## キャンセル

Worker はチャンク境界でキャンセルフラグを確認する。キャンセル時は作成中の一時ファイルを破棄し、保存済み Feature は状態を `partial` として記録する。DB 状態とファイル状態が矛盾しないよう、確定前は temp に書く。

## エラー処理

動画破損、ffprobe 失敗、GPU OOM、CUDA 初期化失敗、保存失敗を区別してログ化する。ユーザー向けには原因、対象ファイル、再試行可能性、推奨対応を表示する。
